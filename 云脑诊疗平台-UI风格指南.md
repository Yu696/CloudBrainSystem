# 医疗信息系统项目 UI 设计规范

## 1. 设计概述
本规范旨在定义医疗信息系统（医生端、管理端、患者端）的统一视觉语言，确保产品在不同端口间具备专业、严谨且人性化的交互体验。

---

## 2. 核心视觉原则
- **专业与严谨 (Professional):** 医疗数据的处理需具备高准确性和清晰度。
- **高效与易用 (Efficient):** 操作路径最小化，关键信息即时可达。
- **关怀与信任 (Empathetic):** 患者端色彩应柔和，建立信任感。

---

## 3. 色彩体系
| 类型 | 用途 | 色值 (示例) |
| :--- | :--- | :--- |
| **主色 (Primary)** | 品牌色、高亮操作 | `#007AFF` (医疗蓝) |
| **辅助色 (Secondary)** | 状态提示、图标 | `#34C759` (绿-正常), `#FF3B30` (红-紧急) |
| **中性色 (Neutral)** | 背景、文字、边框 | `#F4F7FE` (背景), `#1C1C1E` (深色文字) |

---

## 4. UI 风格分区规范

### 4.1 医生端 (Doctor's Portal)
- **布局:** 左侧导航 + 顶部工具栏 + 中部工作区。
- **视觉重点:**
  - 采用 **卡片式布局 (Card Layout)** 区分患者信息、排班和病历记录。
  - 文字对比度高，确保医生快速阅读病历。
  - 状态标签采用圆形圆角设计，区分问诊状态。

### 4.2 管理端 (Hospital Admin Dashboard)
- **布局:** 侧边导航栏 (深色系) + 右侧多模块数据看板。
- **视觉重点:**
  - **数据可视化:** 引入趋势折线图、柱状图，体现数据掌控感。
  - **表格设计:** 清晰的列间距，支持排序与状态筛选。
  - **背景色:** 采用深色侧边栏提升系统庄重感。

### 4.3 患者端 (Patient Mobile App)
- **布局:** 底部标签导航 + 大圆角卡片。
- **视觉重点:**
  - **亲和力:** 柔和背景色，大量留白减少焦虑感。
  - **交互:** 突出预约、查询等核心行动按钮 (CTA)，圆角半径增大，提升触控点击感。

---

## 5. 组件规范 (UI Components)
1.  **字体:** 采用系统默认无衬线字体 (如苹方或 Segoe UI)，字重区分层级。
2.  **阴影:** 采用极其轻微的软阴影 (`shadow-sm`)，营造悬浮感但不干扰阅读。
3.  **圆角:** 全局采用统一圆角值 (建议：容器 `8px` - `12px`)。
4.  **图标:** 使用扁平、线性图标，粗细一致。

---

## 6. 开发实施建议
- 建议采用统一的设计系统库 (如 Ant Design 或 Material Design) 进行二次定制，确保各端间距、行高、文字大小的严格一致。
