# 云脑诊疗平台 — 微服务架构方案

> 整合自《东软智慧云脑诊疗平台实践任务描述》与《第31组-20246136-余润东-云脑医疗-详细设计说明书》

---

## 一、技术栈

| 层次 | 技术选型 |
|------|----------|
| 前端 | Vue 3 + TypeScript + Vite + Element Plus + Axios + Vue Router + Pinia |
| 后端 | Spring Boot 3 + Spring MVC + Spring Data JPA + Knife4j (Swagger3) |
| 数据库 | MySQL |
| AI 能力 | HTTP API 调用大模型服务 + 知识图谱服务 |
| 认证 | JWT |
| 通信 | RESTful API |
| 设计风格 | 医疗蓝 (#1A73E8) + 洁净白 (#FFFFFF)，思源黑体 / Roboto，响应式 |

## 二、项目模块

### 2.1 模块总览（8 大模块）

| 编号 | 模块名称 | 子系统 | 核心职责 |
|------|---------|--------|---------|
| M01 | 用户中心模块 | 用户中心子系统 | 医生/管理员/患者注册登录、JWT 认证、角色权限 (RBAC) |
| M02 | 患者档案模块 | 患者管理子系统 | 患者建档、信息查询更新、身份证校验、二维码生成 |
| M03 | 诊疗记录模块 | 门诊医生智能工作台 | 病历 CRUD、处方开具、检查单开具、检查报告查看、数据统计 |
| M04 | AI 辅助诊断模块 | AI 诊断子系统 | 症状分析、影像 AI 识别、科室推荐、疾病匹配、置信度计算 |
| M05 | 影像管理模块 | 影像管理子系统 | DICOM 上传/预览/标注/对比、格式转换、存储管理 |
| M06 | 预约管理模块 | 预约子系统 | 挂号/取消/改约、医生排班、时段锁定、支付、号源池管理 |
| M07 | 药库管理模块 | 药库管理子系统 | 药品搜索/录入/修改/删除、库存管理、发药、取药单打印 |
| M08 | 系统管理模块 | 系统管理子系统 | 用户增删改查、角色绑定、权限配置、状态启禁 |

### 2.4 业务流程

```
患者注册登录 → 智能分诊(输入症状) → AI推荐科室医生 → 在线挂号 → 支付
    → 医生接诊(查看患者档案) → 录入对话记录 / 开检查单
    → AI生成病历(人工确认) → 保存病历
    → 开具处方 → AI审核处方(风险提示) → 提交处方
    → 药库发药 → 打印取药单 → 患者取药
```

### 3.3 服务间调用关系

```
patient-service      ──(Feign)──► user-service          查询/校验患者关联用户
appointment-service  ──(Feign)──► ai-service            挂号前调用智能分诊
appointment-service  ──(Feign)──► user-service          查询医生信息 & 排班
appointment-service  ──(Feign)──► patient-service       查询患者信息
medical-service      ──(Feign)──► ai-service            开方调用处方审核、对话转病历
medical-service      ──(Feign)──► patient-service       查询患者档案
medical-service      ──(Feign)──► image-service         关联检查影像
medical-service      ──(Feign)──► pharmacy-service      开具处方时校验药品库存
ai-service           ──(Feign)──► image-service         获取待分析影像
ai-service           ──(Feign)──► notification-service  高风险告警推送
pharmacy-service     ──(Feign)──► medical-service       获取处方信息
pharmacy-service     ──(Feign)──► notification-service  库存预警推送
```

### 3.4 数据库拆分

| 服务 | 数据库 | 核心表 |
|------|--------|--------|
| user-service | cloudbrain_user | user, role, permission, user_role, system_user, login_log |
| patient-service | cloudbrain_patient | patient, qr_code |
| appointment-service | cloudbrain_appointment | appointment, schedule, time_slot, department, payment |
| medical-service | cloudbrain_medical | medical_record, prescription, prescription_item, examination_order, audit_record |
| ai-service | cloudbrain_ai | diagnosis_result, triage_log, generation_log, prompt_template, disease_kb |
| image-service | cloudbrain_image | medical_image, annotation, storage_config, conversion_log |
| pharmacy-service | cloudbrain_pharmacy | drug, stock, ship_record, warehouse |
| notification-service | cloudbrain_notify | notification, websocket_session, alert_rule |

**跨服务数据规则：**
- 各服务只持有自身数据，跨服务用 ID 关联（如挂号记录存 `patient_id`、`doctor_id`）
- 强一致性场景（如挂号成功必须锁定号源）→ Seata AT 分布式事务
- 最终一致性场景（如 AI 审核结果通知）→ RabbitMQ 异步
- 号源锁定使用 Redis 分布式锁（15 分钟超时自动释放）

### 3.5 微服务技术组件

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| 注册中心 | Nacos | 服务发现与健康检查 |
| 配置中心 | Nacos Config | 统一配置管理，动态刷新 |
| API 网关 | Spring Cloud Gateway | 基于 WebFlux，高性能路由 |
| 服务调用 | Spring Cloud OpenFeign | 声明式 HTTP 客户端 |
| 负载均衡 | Spring Cloud LoadBalancer | 客户端负载均衡 |
| 熔断降级 | Sentinel | 流量控制、熔断、系统保护 |
| 分布式事务 | Seata (AT 模式) | 跨服务数据一致性 |
| 分布式锁 | Redis + Redisson | 号源时段锁定 |
| 消息队列 | RabbitMQ | 服务异步解耦 |
| 链路追踪 | SkyWalking | 跨服务调用链追踪 |
| 统一日志 | ELK | 日志聚合与检索 |
| 对象存储 | MinIO | DICOM 影像文件存储 |

### 3.6 AI 服务降级策略

| 场景 | 降级方案 |
|------|---------|
| AI 分诊不可用 | 前端展示全部科室列表，用户手动选择 |
| AI 影像诊断不可用 | 仅展示原始影像，标记"AI 分析暂不可用" |
| AI 处方审核不可用 | 仅做本地规则校验（库存、必填项），标记"未经 AI 审核" |
| AI 病历生成不可用 | 医生手动填写病历表单 |
| AI 服务超时 | Sentinel 熔断，3 秒超时后返回降级响应 |
| 置信度 < 50% | 自动标记"低置信度"，触发人工复核通知 |
