# 云脑诊疗平台 — 数据库设计说明书

> 对应微服务 8 个独立数据库，共计 **40 张表**

---

## 一、cloudbrain_user（用户服务 · M01 + M08）

### 1.1 `user` — 用户表

| 字段名 | 类型(长度) | 约束 | 说明 |
|--------|-----------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| user_id | VARCHAR(32) | UNIQUE, NOT NULL | 用户唯一标识 |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名（登录账号） |
| password | VARCHAR(255) | NOT NULL | BCrypt 加密密码 |
| real_name | VARCHAR(50) | NOT NULL | 真实姓名 |
| phone | VARCHAR(20) | | 手机号 |
| email | VARCHAR(100) | | 邮箱 |
| avatar_url | VARCHAR(255) | | 头像 URL |
| user_type | TINYINT | NOT NULL, DEFAULT 0 | 类型：0=医生 1=管理员 2=患者 |
| status | TINYINT | NOT NULL, DEFAULT 1 | 状态：0=禁用 1=启用 |
| last_login_time | DATETIME | | 最后登录时间 |
| create_time | DATETIME | NOT NULL, DEFAULT NOW() | 创建时间 |
| update_time | DATETIME | NOT NULL, DEFAULT NOW() ON UPDATE | 更新时间 |
| deleted | TINYINT | DEFAULT 0 | 逻辑删除标记 |

### 1.2 `role` — 角色表

| 字段名 | 类型(长度) | 约束 | 说明 |
|--------|-----------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| role_id | VARCHAR(32) | UNIQUE, NOT NULL | 角色唯一标识 |
| role_name | VARCHAR(50) | NOT NULL | 角色名称（如：主任医生、普通医生、挂号员、药库管理员） |
| role_code | VARCHAR(50) | UNIQUE, NOT NULL | 角色编码 |
| description | VARCHAR(255) | | 角色描述 |
| status | TINYINT | DEFAULT 1 | 状态：0=禁用 1=启用 |
| create_time | DATETIME | NOT NULL, DEFAULT NOW() | 创建时间 |
| update_time | DATETIME | NOT NULL, DEFAULT NOW() ON UPDATE | 更新时间 |

### 1.3 `permission` — 权限表

| 字段名 | 类型(长度) | 约束 | 说明 |
|--------|-----------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| permission_id | VARCHAR(32) | UNIQUE, NOT NULL | 权限唯一标识 |
| permission_name | VARCHAR(50) | NOT NULL | 权限名称 |

### 1.4 `role_permission` — 角色-权限关联表

| 字段名 | 类型(长度) | 约束 | 说明 |

### 1.5 `user_role` — 用户-角色关联表

| 字段名 | 类型(长度) | 约束 | 说明 |

### 1.6 `system_user` — 系统管理用户表（M08）

## 二、cloudbrain_patient（患者服务 · M02）

### 2.1 `patient` — 患者档案表

| 字段名 | 类型(长度) | 约束 | 说明 |
|--------|-----------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| patient_id | VARCHAR(32) | UNIQUE, NOT NULL | 患者唯一标识 |
| user_id | VARCHAR(32) | FK → user(user_id) | 关联用户 ID（可为空，未注册用户） |
| medical_record_no | VARCHAR(32) | UNIQUE, NOT NULL | 病历号（系统生成） |
| name | VARCHAR(50) | NOT NULL | 患者姓名 |
| id_card | VARCHAR(18) | UNIQUE, NOT NULL | 身份证号 |
| gender | TINYINT | NOT NULL | 性别：0=未知 1=男 2=女 |
| birth_date | DATE | NOT NULL | 出生日期 |
| age | INT | GENERATED | 年龄（由 birth_date 计算） |
| phone | VARCHAR(20) | NOT NULL | 联系电话 |
| emergency_phone | VARCHAR(20) | | 紧急联系人电话 |
| address | VARCHAR(255) | | 家庭住址 |
| blood_type | VARCHAR(10) | | 血型：A/B/AB/O/其他 |
| allergy_history | TEXT | | 过敏史（文本描述） |
| genetic_diseases | TEXT | | 遗传病史（文本描述） |
| medical_history | TEXT | | 既往病史（文本描述） |
| qr_code_url | VARCHAR(255) | | 二维码 URL |
| id_card_photo_front | VARCHAR(255) | | 身份证正面照 |
| id_card_photo_back | VARCHAR(255) | | 身份证反面照 |
| source | TINYINT | DEFAULT 1 | 建档来源：1=医院建档 2=在线注册 3=转诊 |
| status | TINYINT | DEFAULT 1 | 状态：0=归档 1=正常 |
| create_time | DATETIME | NOT NULL, DEFAULT NOW() | 创建时间 |
| update_time | DATETIME | NOT NULL, DEFAULT NOW() ON UPDATE | 更新时间 |
| deleted | TINYINT | DEFAULT 0 | 逻辑删除标记 |

> INDEX idx_patient_name(name), INDEX idx_patient_phone(phone)

## 九、数据库关系总图

### 9.1 跨服务 ID 引用关系

```
user-service:
  user ◄──── user_role ◄──── role ◄──── role_permission ◄──── permission
    │
    └──── system_user
    └──── login_log

patient-service:
  patient (独立管理, 通过 patient.user_id 关联 user-service)

appointment-service:
  department ◄──── doctor ◄──── schedule ◄──── time_slot ◄──── appointment ◄──── payment

medical-service:
  medical_record ◄──── prescription ◄──── prescription_item
  medical_record ◄──── examination_order ◄──── examination_result
  prescription ◄──── prescription_audit_log

ai-service:
  (独立服务, 通过 patient_id / doctor_id 等 ID 关联其他服务)
  triage_log
  diagnosis_result
  generation_log ──► prompt_template
  disease_knowledge

image-service:
  medical_image ◄──── annotation
  medical_image ◄──── conversion_log
  storage_config

pharmacy-service:
  drug ◄──── drug_stock ◄──── warehouse
  prescription_id ──(外部引用)──► ship_record
  drug ◄──── stock_alert

notification-service:
  notification (接收人 user_id 引用 user-service)
  websocket_session
  alert_rule
```

### 9.2 主外键约束规则

| 约束类型 | 策略 |
|---------|------|
| 同一数据库内（如 user 内） | 使用 MySQL FOREIGN KEY，级联删除 RESTRICT |
| 跨库引用（如 appointment 引用 patient） | 程序层维护一致性，数据库层不建外键，仅存 ID 做逻辑关联 |
| 逻辑删除 | 所有业务表均包含 `deleted` 或 `status` 字段，不物理删除 |

---

> 数据库命名约定：
> - 数据库名：`cloudbrain_{服务名}`
> - 表名：全小写蛇形命名
> - 主键：统一 `id BIGINT AUTO_INCREMENT`
> - 业务 ID：统一 `VARCHAR(32)` UUID 格式
> - 时间字段：`create_time` / `update_time` 每表必备
> - 字符集：`utf8mb4` / `utf8mb4_unicode_ci`
> - 引擎：`InnoDB`
